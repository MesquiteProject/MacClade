MacClade 4.0 Notes

2 April 1999

MacClade 4.0a9 - PowerPC and 68K versions

These notes cover some of the new aspects of the test version of MacClade 4. For almost all features other than the ones mentioned herein, MacClade 4 works exactly as does MacClade 3.07.

Bug reports to David, beetle@ag.arizona.edu, please; any suggestions for new features also very much welcomed.

This version of MacClade is still a test version. Many of the new features are only partly developed; there are a few screen display problems, especially in the data editor. It is an alpha test version, which means that it does not have all of the features planned for the final release.

There may be some worse bugs. Use with caution. As always with test software, make sure you back up your files before using this version.

In the following notes, known bugs are marked with a .

Notes on features to add or alter are marked with .

General Aspects

PowerPC version

One big change is that there is now a PowerPC native version of MacClade. (MacClade 3.07 is a 68K program, which means it runs relatively slowly on PowerMacs.)

Preferences

For MacClade version 4, the preferences system was completely rewritten. This was necessary in part to make it much easier to update, and in part to make it work under MacOS X. A consequence of this that most previous preferences will not be processed by MacClade, and you will need to reset them.

Closing windows and files

- now, Command-W closes the frontmost window; this will close the file only if it is the last window open in MacClade.

Background operations

Many operations are now backgroundable; that is, you can bring another application to the forefront and let MacClade carry on calculations in the background.

you won't be notified if an error is encountered in MacClade when it is in the background
sometimes when MacClade is calculating in the background it hogs too much processing time.

Menu reorganization & enabling/disabling

Assume and Format menus subsumed into Characters menu. Taxa menu is new; Tree menu is reorganized.

Some menu items are enabled when they should be disabled (grayed).

Data editor

Many changes, as described below.

 

Alignment Tools

See On-the-fly amino acid translation, coloring data cells, block moving tool, split others tool, scissors tool, selection wand, selected block tool; Dim characters; Consensus sequences; Find facility.

If you use these tools, you might wish to choose "Frequent Redrawing" from the Show menu. This will force MacClade to redraw the cells in the editor more often than normal, thus avoiding any of the screen redrawing errors that might be present in this version. Screen redrawing errors used a real problem with earlier developmental version, but is much less of a problem now. In the unlikely event that this version of MacClade will muck up your data, this will allow you to detect such problems more quickly.

Here is a brief description of each tool:

standard editor tool:

more or less same as what is was in 3.07. If another tool is currently selected, you can get back to this tool by holding down the Command key.

block selector:

- selects blocks in the matrix

selection wand:

- touch on a taxon name, selects all taxon names; t
- touch on a character name, selects all character names.
- touch on a cell in the matrix, if it is a gap, then the largest contiguous piece of gaps containing that cell is selected, restricted to one taxon; if it is not a gap, then the largest continuous piece of non-gaps containing that cell is selected.

-if you hold down the Shift key, the selection is extended. In particular, if the previous selection contains non-gaps, then it will select blocks of data bounded by gaps; if the previous selection contains only gaps, then it will select the widest block of gaps for all taxa selected.

The pop-up menu, available by touching on the tool in the tool palette, allows you to choose between various modes of behavior. Here are the modes:

  • normal: selects blocks of single sequences
  • block mode, taxon priority: in one touchs on a gap, then MacClade looks at the character touched upon, and selects as many contiguous cells as possible for that character that contain gaps; it then expands the selection horizontally to the largest size it can to have a rectangle of pure gaps. So first it expands vertically, then horizontally to get a selection of pure gaps. If one touches on a non-gap, then it does the same thing, except that it selects the largest block of pure non-gaps.
  • block mode, character priority: in one touchs on a gap, then MacClade looks at the cell touched upon, and selects as many contiguous cells as possible for that taxon that contain gaps; it then expands the selection vertically to the largest size it can to have a rectangle of pure gaps. So first it expands horizontally, then vertically to get a selection of pure gaps. If one touches on a non-gap, then it does the same thing, except that it selects the largest block of pure non-gaps.
  • mixed block mode: not working very well yet.

If one wants to select an entire sequence, not counting the terminal gaps, then choose the Select Entire Sequence option in the tool's popup menu.

select to end tool:

- drag over some cells, and MacClade will select all cells in those taxa from those characters to the end of the matrix.

- hold down Option, and MacClade will select in the reverse direction, selecting to the start of the matrix

- if you touch on the taxon names, or on the taxon numbers, then it will select all taxon names/ all taxa to the end of the matrix (or to the start, if Option is held down)

scissors tool:

excises pieces of the matrix. it removes the data and moves the remained of those sequences to the left.

It can be applied to previously selected blocks, or it can be dragged to select the cells to be deleted.

For example, if one used it on following selected block:

those 12 cells would be removed from the matrix and the end six sequences would be shifted to the left:

If you hold down the Option key when using the scissors tool, then the start of the sequences are shifted to the right to fill in the deleted block.

block move tool:

- main alignment tool. to use it touch on a nucleotide or amino acid; MacClade will automatically grab and let you move an entire chunk of the sequence (by an entire chunk, we mean the largest contiguous piece of non-gaps). You will be able to move it forward and backward through any bounding gaps; you should not be able to move it over any nucleotides or amino acids.

the ends of this block are sticky, in that if you move it against some other nucleotides, then release the mouse button, then click again, the block will be expanded to include those additional nucleotides.

This tool allows you to move only one sequence at a time.

I have added two checks to make sure this tool is not damaging your data. Before moving data, this tool does a check to make sure no data will be accidentally overwritten. After moving data, it checks to see if the frequencies of nucleotides were changes with the move. If this does change, it means MacClade did something wrong, and it will warn you.

The popup menu for this tool has options that alter its behavior. In particular:

  • if Allow Matrix Edges to Expand is checked, then MacClade will add more characters to the start or end of the matrix to make room for sequences pushed off the start or end.
  • if Move Entire Sequence is checked, then the entire sequence, exclusive of terminal gaps, will be moved
  • If Accumulate Sticky Blocks is checked, then as you move one piece along, if it bumps into another piece, then that other piece will be joined to the first piece and you will now be moving both pieces, as one.
  • if Allow Block Splitting is checked, then the Block Move Tool behaves as the Split Block Tool when the mouse is over the boundary between two cells

 as with some of the other alignment tools, if in using them you force MacClade to scroll the matrix, there may be some screen display problems; in particular, visual remnants of old block positions may still exist.

selected block tool:

an alignment tool that lets you move selected blocks in the matrix. Equivalent, more or less, to the block move tool, except that it only works on selected blocks, and doesn't allow you to split a block. The advantage is that it allows you to move multiple sequences at once. Valuable in conjunction with the magic wand tool.

can drag this tool through unselected regions to select them.

splitblock tool:

- another alignment tools. allows you to split a block of data to introduce gaps. must be used on boundaries between cells.

split others tool:

Similar to the split block tool except that it moves all sequences OTHER than the one being dragged, and currently it moves not just one piece of data on one side by the entire sequence on that side.

When one clicks on the boundary between two characters in one taxon and pulls to the right or left, this tool moves all the OTHER taxa to the right or left. For example, if one positioned the tool as shown:

and pulled one column to the right, all other sequences would be moved to the right, and gaps inserted:

stamp clipboard tool. Select a block of cells, and copy them to the clipboard. Then choose this tool, and you will see be able to move the clipboard contents around the edtior. To paste the contents onto a region, click the mouse button to stamp the cells down.

pop-up states tool. touch this on a cell, and a menu will pop up listing the states for that character. choose one, and that state will be entered into that cell.

can also be used to fill all cells in a selected block.

can also be used to fill in some default names for a selected set of taxon names.

paint bucket tool. use this tool on a selected block of cells and that block will be filled with the state represented by the "paint" (see the eyedropper tool, below, for a description of the paint options and how one changes the paint).

- can also drag this tool over an unselected block of cells to select them and paint them.

Modes, available in popup menu:

Fill with Paint: the normal mode, in which the cells are filled with the paint.
Fill with Random: cells are filled with random data, in the frequencies set in the Fill Random dialog box in the Utilities menu
Fill with Observed States: cells are filled with only the states ALREADY PRESENT in each character, in equal frequencies. Thus, if you used that on character 2, with states 0 and 1, then the cells would be filled with 0 or 1, each with a probability of 0.5; if you used it on character 5, with states 0,1,and 3, then the cells of that character would be filled with states 0,1, or 3, each with a probability of 0.3333.
Fill with Observed Frequencies: cells are filled with only the states ALREADY PRESENT in each character, with probabilities proportional to the observed frequencies. Thus, if you used that on character 2, which had two taxa with state 0 and 8 with state 1, then the cells would be filled with 0 with a probability of 0.2, and 1 with a probability of 0.8.

Restrict Fill option:

This option (available in the popup menu) lets you restrict the fill, by presenting you with a dialog box containing options. If you ask it to fill only cells with state X, then MacClade will ignore all other cells in the selected area, and only fill those with state X. Choosing "not (? or -)" from the popup menu in the dialog box will tell MacClade to only fill cells containing data.

You can also specify what proportion of the cells to fill. If you choose 0.1, then MacClade will fill each of the chosen cells with a probability of 0.1; thus, on average, only 10% of the cells will be altered.

For example, if you asked MacClade to fill only cells with missing data, and asked it to fill 0.5 of the cells with the observed frequencies, then MacClade would fill 50% of the missing data with states present in each character, and fill them in the observed frequencies. This would allow you to investigate the effects of missing data.

You could also investigate the effects of errors on your data by, say, filling 0.001 of the cells containing data with random data.

paint brush tool. use this tool to set the state of any cell touched on to be the state represented by the "paint" (see the eyedropper tool, below, for a description of the paint options and how one changes the paint).

eyedropper tool:

- touch on a cell in the matrix and MacClade will use the state in that cell as the paint state.
- touch on a taxon name or a taxon number and the paint state will be the state present in that taxon. e.g. say taxon 5's number is touched on, and the states for taxon 5 are 01102 for the first five characters. Then using the paintbrush or paintbucket on any taxon's character 1 will paint state 0, any taxon's character 2 will paint state 1, and so on.

you can also touch on the eyedropper tool in the palette and a little pop-up menu will appear allowing you to choose the paint state.

 

Consensus Sequences

Choose Consensus Sequence from the Display menu to ask MacClade to automatically calculate Consensus Sequences. You can add as many as you wish, and have them of different stringencies. Once added, you can reorder them by dragging, and you can delete them by selecting them and hiting the Delete key.

You can also move them to be true members of the matrix by selecting one of them and choosing "Transfer Consensus to Taxon" from the Taxa menu.

If you choose a taxon set in the Consensus Taxa submenu of the Display menu, then the consensus sequences will be calculated only over elements of that taxon set. By default the consensus is calculated over all taxa. You must first create other taxon sets to use them in this way (do this by selecting rows in the Taxon List window, and choosing Store Taxon Set from the Taxon Sets submenu of the Taxa menu).

When you use the Block Move tool on a piece, and if you chose "show matches during block move" in the Consensus Sequences dialog box, the box with the name of the consensus sequence changes to have in it the total number of elements (undimmed/dimmed) in the consensus sequence.

Selecting runs of gaps

In the selection submenu in the Character List window, if you choose "Gap Runs..." you will be presented with a dialog box, which will allow you to select in the Character List window all characters that participate in runs of gaps of defined lengths. This does not include terminal runs of gaps, at either end of a sequence. For example, in the following matrix, if you asked to select characters participating in gap runs of length 2 or more, you would select characters 3-5, 10-14, 18-19, as shown in the row marked "gap run of 2". If you increase the length of the gap runs desired, fewer and fewer characters would be selected.

In general, if you ask for gap runs of length x, then if there is, in any taxon, a string of contiguous gaps of length greater than or equal to x, and imagine that string of gaps went from character y to character z, then characters y through z would be selected.

NEXUS Block management

MacClade files are written in the NEXUS format, a file format shared with PAUP and other programs. NEXUS files consist of a sequence of blocks of information. For example, a data file might include a DATA block, an ASSUMPTIONS block, a TREE block, and so on.

MacClade looks after the creation of all of the blocks containing information it processes. For example, when you create a data file in MacClade, and ask MacClade to save it, it writes into the file a block (perhaps called DATA) that contains the data matrix, names of characters, etc., written in NEXUS commands. If the only program you ever use is MacClade, then you can let MacClade do all of the reading and writing of NEXUS blocks for you, without ever worrying about them. However, if you want to use the file in other programs, such as PAUP, it can be handy or necessary to learn how to write NEXUS blocks yourself.

This version of MacClade now allows you to create additional NEXUS blocks and edit them from within MacClade itself. To create a new NEXUS block, choose New NEXUS Block from the NEXUS Block submenu of the Edit menu. You will be presented with the NEXUS Blocks List window, showing you all of the NEXUS blocks MacClade understands and has control over (these are yellowed, and are not currently editable - this will change slightly soon), as well as the new, untitled block you just created. Type in the appropriate name for the block. You can enter things into the block by double-clicking on the block's number in this list window, or by selecting the block's row and choosing "Edit NEXUS block" from the NEXUS Blocks submenu.
